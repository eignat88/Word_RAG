# Jira Breakdown: Epic и задачи для RAG-базы знаний по Word

## Epic
**KB-RAG-EPIC-1**: Локальная база знаний по Word-документам с семантическим поиском и генерацией ответов.

---

## Этап 1. Инфраструктура и БД

### KB-RAG-1: Поднять PostgreSQL + pgvector
**Описание**
- Развернуть локальный Postgres.
- Установить расширение `pgvector`.
- Настроить подключение из Python-сервисов.

**Критерии приемки**
- БД поднимается командой из README.
- `CREATE EXTENSION vector;` выполняется успешно.
- Сервис может выполнить тестовый `SELECT 1`.

### KB-RAG-2: Создать схему knowledge_base
**Описание**
- Создать таблицу `knowledge_base`.
- Добавить индексы для фильтрации (`fd_number`, `section`).
- Подготовить миграцию.

**Критерии приемки**
- Миграция воспроизводимо создает схему.
- Таблица содержит поля: `id`, `document_name`, `fd_number`, `section`, `chunk_text`, `embedding`.

---

## Этап 2. Инжест документов

### KB-RAG-3: Реализовать парсер .docx
**Описание**
- Прочитать документы из указанной папки.
- Извлечь текст и заголовки разделов.
- Нормализовать служебные символы.

**Критерии приемки**
- Обрабатывается пакет из 100+ файлов.
- Для каждого документа извлекаются секции.

### KB-RAG-4: Реализовать чанкинг по секциям
**Описание**
- Делить текст на чанки 300–1000 символов.
- Не смешивать разделы в одном чанке.
- Сохранять метаданные документа и секции.

**Критерии приемки**
- Ни один чанк не содержит данные из двух разных разделов.
- Средний размер чанка в требуемом диапазоне.

### KB-RAG-5: Повторная индексация (upsert)
**Описание**
- Добавить механизм переиндексации документов.
- Заменять устаревшие чанки при повторной загрузке.

**Критерии приемки**
- Повторный запуск не создает дубликатов.
- Обновления документа отражаются в поиске.

---

## Этап 3. Embeddings и поиск

### KB-RAG-6: Интеграция локальных embeddings через Ollama
**Описание**
- Подключить embedding-модель через локальный endpoint Ollama.
- Добавить батчевую генерацию embeddings.

**Критерии приемки**
- Для каждого чанка в БД присутствует embedding.
- Ошибки Ollama логируются, есть retry/pause политика.

### KB-RAG-7: Реализовать векторный поиск top-N
**Описание**
- Преобразовывать вопрос в embedding.
- Выполнять similarity-поиск в `pgvector`.
- Возвращать фрагменты с метаданными.

**Критерии приемки**
- Поддерживается `LIMIT N`.
- Возвращаются `document_name`, `fd_number`, `section`, `chunk_text`.

### KB-RAG-8: Добавить фильтры по fd_number и section
**Описание**
- Поиск с фильтрами.
- Комбинируемые условия фильтрации.

**Критерии приемки**
- `fd_number = DAX-7407` ограничивает поиск только этим ФД.
- Фильтр по `section = Алгоритм` работает корректно.

---

## Этап 4. Генерация ответов

### KB-RAG-9: Реализовать RAG prompt-assembler
**Описание**
- Собирать контекст из топ-N чанков.
- Формировать промпт «Ответь строго на основе контекста».

**Критерии приемки**
- Ответ содержит ссылки на источники.
- При пустом контексте возвращается контролируемое сообщение.

### KB-RAG-10: Интегрировать LLM через Ollama
**Описание**
- Отправлять собранный промпт в локальную LLM.
- Возвращать ответ и список использованных источников.

**Критерии приемки**
- Среднее время ответа ≤ 10 сек на типовом окружении.
- Наблюдаемый процент галлюцинаций снижается при наличии контекста.
